# Claude Code 源码核心功能深度分析

## 执行摘要

本文档是对 Claude Code 源码进行深度分析后的核心发现总结，旨在为复现工作提供技术指导和架构参考。

**分析范围**:
- 代码规模：~1,900 TypeScript 文件，512,000+ 行代码
- 核心模块：工具系统、命令系统、查询引擎、服务层
- 技术栈：Bun + TypeScript + React/Ink + Anthropic SDK

  // 状态初始化
  this.discoveredSkillNames.clear();
  const startTime = Date.now();

  // 构建系统提示
  const { defaultSystemPrompt, userContext } = await fetchSystemPromptParts({...});

  while (true) {

    const response = await this.callAPI({
      messages: this.mutableMessages,
      system: defaultSystemPrompt,
      tools: this.getToolDefinitions(),
    });

      yield this.normalizeEvent(event);

    if (toolUses.length === 0) break;

    // 执行工具
    const results = await this.executeTools(toolUses);
    this.mutableMessages.push(...results);
  }
}

const TOOL_DEFAULTS = {
  isEnabled: () => true,
  isConcurrencySafe: () => false,

**权限模式**:
| 模式 | 行为 |
|------|------|
| `default` | 危险操作询问，只读操作允许 |
| `auto` | 仅允许只读操作，其他全部拒绝 |
| `plan` | 所有操作都需要询问 |
| `bypass` | 所有操作直接允许 |

### 2.5 MCP 集成：协议适配

**MCP 客户端架构**:
```typescript
class MCPManager {
  private connections: Map<string, MCPServerConnection> = new Map();

  // 连接 MCP 服务器
  async connect(config: McpServerConfig): Promise<MCPServerConnection> {
    // 1. 创建传输层 (stdio/sse/websocket)
    const transport = await this.createTransport(config);

    // 2. 创建 MCP 客户端
    const client = new Client(
      { name: 'claude-code', version: '1.0.0' },
      { capabilities: {} }
    );
    await client.connect(transport);

    // 3. 获取工具和资源
    const tools = await client.listTools();
    const resources = await client.listResources();

    // 4. 包装为内部 Tool 格式
    const wrappedTools = tools.map(toolDef =>
      this.createMcpTool(config.name, toolDef)
    );

    const connection = {
      name: config.name,
      client,
      tools: wrappedTools,
      resources,
      status: 'connected',
    };

    this.connections.set(config.name, connection);
    return connection;
  }

  // 创建 MCP 工具包装
  private createMcpTool(serverName: string, toolDef: MCPToolDefinition): Tool {
    return buildTool({
      name: `mcp__${serverName}__${toolDef.name}`,
      description: toolDef.description,
      inputJSONSchema: toolDef.inputSchema,
      isMcp: true,
      mcpInfo: { serverName, toolName: toolDef.name },

      async call(input, context) {
        const client = context.mcpClients.find(c => c.name === serverName);
        if (!client) throw new Error(`MCP server "${serverName}" not found`);
        return await client.callTool(toolDef.name, input);
      },
    });
  }
}
```

const DANGEROUS_PATTERNS = [
  /^rm\s+(-[rf]+\s+)?\//,           // rm -rf /

// FileReadTool.tsx
async function validatePath(requestedPath: string, cwd: string): Promise<string> {
  // 解析为绝对路径
  const fullPath = path.resolve(cwd, requestedPath);

  // 检查是否在允许目录内
  const allowedDirs = getAdditionalWorkingDirectories();
  const isInAllowedDir = allowedDirs.some(dir =>
    fullPath.startsWith(dir.path)
  );

  if (!isInAllowedDir) {
    throw new SecurityError(
      `访问被拒绝：${fullPath} 不在允许的工作目录内`
    );
  }

  // 检查符号链接
  const realPath = await fs.realpath(fullPath);
  if (!realPath.startsWith(cwd)) {
    throw new SecurityError('符号链接指向工作目录外');
  }

  return fullPath;
}

## 六、关键技术挑战

### 6.1 流式响应与工具调用的协调

**挑战**: 需要在流式响应中检测工具调用并暂停流式输出，等待工具执行完成后继续

**解决方案**:
```typescript
async function* handleStreamingResponse(stream: APIStream) {
  const toolUseBuffer: ToolUseBlock[] = [];
  let isStreamingToolUse = false;

  for await (const event of stream) {
    if (event.type === 'content_block_start' && event.content_block.type === 'tool_use') {
      isStreamingToolUse = true;
      toolUseBuffer.push(event.content_block);
    } else if (event.type === 'content_block_delta' && isStreamingToolUse) {
      toolUseBuffer[toolUseBuffer.length - 1].input += event.delta.text;
    } else if (event.type === 'content_block_stop' && isStreamingToolUse) {
      isStreamingToolUse = false;
      // 执行工具
      const results = await executeTools(toolUseBuffer);
      yield { type: 'tool_results', results };
    } else {
      yield event;
    }
  }
}
```

### 6.2 并发工具执行的控制

**挑战**: 某些工具不能并发执行（如文件写入），需要实现互斥锁

**解决方案**:
```typescript
class ToolExecutor {
  private writeLocks: Map<string, Promise<void>> = new Map();

  async executeTool(tool: Tool, input: unknown): Promise<ToolResult> {
    // 检查是否需要互斥执行
    if (!tool.isConcurrencySafe?.(input as any)) {
      const resourceKey = this.getResourceKey(tool, input);

      // 等待现有锁释放
      while (this.writeLocks.has(resourceKey)) {
        await this.writeLocks.get(resourceKey);
      }

      // 获取锁
      let releaseLock: () => void;
      const lockPromise = new Promise<void>(resolve => {
        releaseLock = resolve;
      });
      this.writeLocks.set(resourceKey, lockPromise);

      try {
        return await tool.call(input as any, this.context);
      } finally {
        releaseLock!();
        this.writeLocks.delete(resourceKey);
      }
    }

    // 并发安全工具直接执行
    return await tool.call(input as any, this.context);
  }

  private getResourceKey(tool: Tool, input: unknown): string {
    // 对于文件操作，使用文件路径作为锁键
    if ('file_path' in (input as any)) {
      return `${tool.name}:${(input as any).file_path}`;
    }
    return tool.name;
  }
}
```

## 七、总结

Claude Code 展示了一个生产级 AI 代理 CLI 系统的完整架构：

### 核心优势

1. **模块化设计** - 工具、命令、服务层清晰分离，易于扩展
2. **类型安全** - 全面的 TypeScript 类型定义，编译时错误检测
3. **性能优化** - 并行预加载、懒加载、缓存策略
4. **可扩展性** - 插件系统、技能系统、MCP 协议
5. **可靠性** - 错误重试、优雅降级、多层权限控制

### 复现关键点

1. **从简单开始** - 先实现核心工具和 API 集成，再逐步扩展
2. **统一接口** - 使用构建器模式统一工具创建
3. **权限优先** - 权限系统应该在早期设计，而非事后添加
4. **测试驱动** - 为每个工具编写单元测试
5. **文档沉淀** - 记录架构决策和实现细节
