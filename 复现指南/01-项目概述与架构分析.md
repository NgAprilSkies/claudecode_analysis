# Claude Code 复现指南

## 第一部分：项目概述与架构分析

### 1.1 项目简介

Claude Code 是 Anthropic 官方的终端 CLI 工具，用于通过自然语言与 Claude 模型交互来执行软件工程任务。

**核心能力：**
- 文件读写和编辑
- Shell 命令执行
- 代码库搜索（Grep/Glob）
- 多代理协作
- 网络搜索和内容抓取
- 会话管理和持久化

**技术栈概览：**

| 层级 | 技术 |
|------|------|
| 运行时 | Bun |
| 语言 | TypeScript (严格模式) |
| 终端 UI | React + Ink |
| CLI 框架 | Commander.js |
| Schema 验证 | Zod v4 |
| API | Anthropic SDK |
| 特性开关 | GrowthBook + bun:bundle |

### 1.3 核心模块详解

#### 1.3.1 入口点 (main.tsx)

**职责：** 应用启动、CLI 解析、渲染器初始化

**关键流程：**
```
启动
  │
  ├─→ 并行预加载
  │    ├─ MDM 设置读取 (startMdmRawRead)
  │    └─ Keychain 预取 (startKeychainPrefetch)
  │
  ├─→ 模块初始化
  │    ├─ 应用安全配置
  │    ├─ 优雅关闭处理
  │    └─ 特性标志检查
  │
  ├─→ Commander.js CLI 解析
  │    └─ 解析命令行参数
  │
  └─→ React/Ink 渲染
       └─ 启动 REPL 循环
```

**关键代码模式：**
```typescript
// 并行预加载优化
profileCheckpoint('main_tsx_entry');
startMdmRawRead();  // 并行读取系统设置
startKeychainPrefetch();  // 并行预取认证凭据

// 特性标志驱动的死代码消除
const voiceCommand = feature('VOICE_MODE')
  ? require('./commands/voice/index.js').default
  : null;
```

#### 1.3.2 查询引擎 (QueryEngine.ts)

**职责：** 管理 LLM 对话生命周期、工具调用循环、Token 预算

**核心状态：**
- `mutableMessages`: 会话消息历史
- `permissionDenials`: 权限拒绝记录
- `totalUsage`: Token 使用统计
- `readFileState`: 文件读取缓存

**查询循环流程：**
```
submitMessage(prompt)
  │
  ├─→ 构建系统提示
  ├─→ 调用 LLM API (流式)
  ├─→ 处理响应
  │    ├─ 如果有 tool_use → 执行工具
  │    ├─ 检查权限
  │    └─ 收集结果
  ├─→ 递归处理直到完成
  └─→ 记录会话历史
```

#### 1.3.3 工具系统 (Tool.ts)

**工具接口定义：**
```typescript
export type Tool<Input, Output, Progress> = {
  name: string;
  inputSchema: z.ZodType<Input>;
  outputSchema?: z.ZodType<Output>;

  // 核心执行
  call(
    args: Input,
    context: ToolUseContext,
    canUseTool: CanUseToolFn,
    parentMessage: AssistantMessage,
    onProgress?: ToolCallProgress<Progress>
  ): Promise<ToolResult<Output>>;

  // 权限检查
  checkPermissions(
    input: Input,
    context: ToolUseContext
  ): Promise<PermissionResult>;

  // UI 渲染
  renderToolUseMessage(
    input: Partial<Input>,
    options: { theme: ThemeName; verbose: boolean }
  ): React.ReactNode;

  renderToolResultMessage(
    content: Output,
    progressMessages: ProgressMessage<Progress>[],
    options: { theme: ThemeName; verbose: boolean }
  ): React.ReactNode;

  // 元数据
  isConcurrencySafe(input: Input): boolean;
  isReadOnly(input: Input): boolean;
  isDestructive?(input: Input): boolean;
  isEnabled(): boolean;

  // 更多可选方法...
}
```

**工具构建器模式：**
```typescript
const TOOL_DEFAULTS = {
  isEnabled: () => true,
  isConcurrencySafe: () => false,
  isReadOnly: () => false,
  isDestructive: () => false,
  checkPermissions: (input) => ({ behavior: 'allow', updatedInput: input }),
};

export function buildTool<D>(def: D): BuiltTool<D> {
  return {
    ...TOOL_DEFAULTS,
    userFacingName: () => def.name,
    ...def,
  };
}
```

#### 1.5.4 责任链模式 (工具编排)

```typescript
// toolOrchestration.ts
async function runTools(toolUses, context) {
  for (const toolUse of toolUses) {
    // 1. 查找工具
    const tool = findToolByName(tools, toolUse.name);

    // 2. 验证输入
    const validation = await tool.validateInput(input, context);
    if (!validation.result) continue;

    // 3. 权限检查
    const permission = await canUseTool(tool, input, context);
    if (permission.behavior !== 'allow') continue;

    // 4. 并发安全处理
    if (!tool.isConcurrencySafe(input)) {
      await runExclusively(toolUse, tool, input, context);
    } else {
      // 5. 并行执行
      promises.push(executeTool(toolUse, tool, input, context));
    }
  }

  return await Promise.all(promises);
}
```

  private cache = new LRUCache({ max: 1000 });

  async get(path: string): Promise<FileState> {
    if (this.cache.has(path)) {
      return this.cache.get(path);
    }
    const state = await readFileState(path);
    this.cache.set(path, state);
    return state;
  }

### 1.7 错误处理机制

#### 1.7.1 API 重试策略

```typescript
// 可重试错误分类
const retryableErrors = [
  'rate_limit',
  'api_timeout',
  'network_error',
  'server_error',
];

async function withRetry<T>(fn: () => Promise<T>): Promise<T> {
  let lastError: Error;
  for (let attempt = 0; attempt < MAX_RETRIES; attempt++) {
    try {
      return await fn();
    } catch (error) {
      lastError = error;
      if (!isRetryable(error)) throw error;
      await exponentialBackoff(attempt);
    }
  }
  throw lastError;
}
```
